== Summary

[1–2 sentences describing the vulnerability in plain language.
Focus on impact, not technical details.]

.Example:
A remote code execution vulnerability exists in the Anvil verification
pipeline that allows an attacker to bypass the permission system and execute
arbitrary shell commands.

== Description

[Detailed technical description of the vulnerability.
Include: what goes wrong, why it goes wrong, and what an attacker can achieve.]

.Example:
The Anvil engine's `VerifyPipeline` processes file paths without
sufficient sanitization. When a crafted task description includes a
path traversal sequence (e.g., `../../../etc/passwd`), the engine
resolves it outside the project sandbox, allowing read access to
arbitrary files on the host system.

This vulnerability can be exploited by any user who can submit a task
to the Anvil engine, including through the CLI, API, or TUI interface.

== Impact

[Describe the real-world impact. Who is affected? What can happen?]

.Example:
* ** Confidentiality:** Attackers can read any file accessible to the Anvil
  process, potentially including configuration files with API keys,
  environment variables with secrets, and source code from other projects.
* **Integrity:** No direct integrity impact; attackers cannot modify files
  outside the sandbox.
* **Availability:** No availability impact.

== Affected Versions

[Specify exactly which versions are affected.]

.Example:
* Anvil v0.1.0 through v0.1.4 — **Affected**
* Anvil v0.2.0+ — **Not affected** (sandbox enforcement added in v0.2.0)
* VerifyLoop all versions — **Not affected** (doesn't process file paths)
* Other FableForge projects — **Not affected**

== Mitigation

[What users should do immediately, before upgrading.]

.Example:
Until you can upgrade, mitigate this vulnerability by:

1. **Restrict file access:** Run Anvil inside a container or VM with limited
   file system access:
+
[source,bash]
....
docker run --read-only --tmpfs /tmp -v $(pwd):/workspace:ro anvil:latest
....

2. **Disable network access:** Prevent exfiltration of read files:
+
[source,bash]
....
anvil run --offline "your task here"
....

3. **Audit logs:** Check Anvil session logs for unusual file read patterns:
+
[source,bash]
....
anvil log --filter "path traversed outside sandbox"
....

== Solution

[Describe the fix. Include the commit hash or PR link.]

.Example:
Upgrade to Anvil v0.2.0 or later, which includes the fix in commit
`abc123def456` (https://github.com/KingLabsA/anvil/pull/123).

The fix adds path canonicalization and boundary checking to the
`VerifyPipeline.resolve_path()` method, ensuring all file operations
stay within the project sandbox.

Upgrade instructions:
+
[source,bash]
....
pip install --upgrade anvil-agent>=0.2.0
....
